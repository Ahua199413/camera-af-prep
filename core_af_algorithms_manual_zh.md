# 核心自動對焦 (AF) 演算法與控制系統手冊 (通用版)

本手冊不針對特定的硬體特性（如可變光圈），專注於**所有現代智慧型手機相機系統中必備的通用核心自動對焦演算法**。這些演算法是任何資深 AF 演算法工程師在技術面試中都必須掌握的底層學理。

---

## 1. 場景變化檢測演算法 (Scene Change Detection, SCD)
*   **目的：** 當使用者移動手機或畫面中的物體移動時，系統必須即時偵測並觸發重新對焦（Re-trigger）。
*   **核心觸發機制與演算法：**

### ① 陀螺儀與加速度計融合 (IMU/Gyro Integration)
*   **原理：** 讀取手機內建陀螺儀（Gyroscope）的角速度數據，進行時間積分計算出手機的旋轉角度。
*   **演算法邏輯：**
    $$\Delta \theta = \int_{t}^{t+\Delta t} \omega(t) dt$$
    當累計旋轉角度 $\Delta \theta$ 大於閾值（通常為 $1.5^\circ \sim 3.0^\circ$）時，判定手機發生晃動，相機進入「場景變更狀態」，觸發重新對焦。

### ② 亮度與直方圖變化量 (Luma / Histogram Change)
*   **原理：** 統計每幀影像的平均亮度（Luma）或 RGB 直方圖的分布變化。
*   **演算法邏輯：**
    比較目前影格 $F_k$ 與前一影格 $F_{k-1}$ 的直方圖交集（Histogram Intersection）。如果重合度低於 $85\%$，或平均亮度變化 $\Delta Y$ 超過預期閾值，代表畫面內容發生劇烈改變。

### ③ 離焦狀態變化 (Defocus Change)
*   **原理：** 即使手機沒動，物體靠近也會導致失焦。此時透過持續讀取 PDAF 視差（Disparity）或 ToF 距離。
*   **演算法邏輯：**
    若連續 3 幀讀取到的 PDAF 相位差均大於設定之安全區範圍（Dead Zone），即使馬達未動，也代表場景主體改變，必須重啟對焦。

## 2. 馬達遲滯與反向間隙補償演算法 (Hysteresis & Backlash Compensation)
*   **物理背景：** 
    聲音線圈馬達（VCM）內部的彈片與鏡頭載體存在滑動摩擦力。當鏡頭**改變移動方向**時（例如：從往 macro 移動改為往 infinity 移動），馬達電流的改變與鏡頭實體位移之間會存在一個物理空隙，這稱為 **Backlash (反向間隙)** 或 **Hysteresis (遲滯現象)**。
*   **解決方案演算法 (Overshoot & Pull-back)：**
    為確保鏡頭不論從哪一個方向出發，最後都能停在完全相同的實體位置，必須使用補償演算法：
    *   **規則：** 統一馬達的「最終 settles 方向」（例如：一律由近往遠 settles）。
    *   **邏輯：** 如果目標是向反方向移動到位置 $DAC_{target}$，演算法會先驅動馬達前進到 $DAC_{target} - DAC_{backlash}$（過衝），然後再向正方向拉回至 $DAC_{target}$。這樣可以完全消除機械摩擦力帶來的滯後誤差。

```
鏡頭實體位置
  ^
  |               / [正向移動路徑]
  |              /
  |             /    <-- 存在 Hysteresis 差值
  |            /
  |           /  / [反向移動路徑]
  |          /  /
  +-----------------------------------> 馬達控制電流 (DAC)
```

## 3. PDAF 數據濾波與置信度評估演算法 (PDAF Filtering & Confidence)
*   **背景：** 影像感測器輸出的原始相位差數據 (Raw PDAF Disparity) 含有高頻隨機雜訊，直接使用會導致馬達微幅抖動，影響錄影畫面穩定度。
*   **濾波與評估演算法：**

### ① 一階無限脈衝響應濾波 (1st-Order IIR Filter)
*   **原理：** 使用指數加權移動平均，平滑連續格之間的對焦目標。
*   **演算法公式：**
    $$DAC_{smooth}(k) = \alpha \cdot DAC_{measured}(k) + (1 - \alpha) \cdot DAC_{smooth}(k-1)$$
    *   $\alpha$ 為濾波係數（$0 < \alpha \le 1$）。
    *   **動態 $\alpha$ 設計：** 當量測值與當前位置差值很大時，使 $\alpha$ 接近 1.0（快速收斂）；當接近目標時，使 $\alpha$ 接近 0.1（防止抖動）。

### ② 異常值剔除門檻 (Outlier Rejection)
*   計算連續格 PDAF 數據的標準差或差值。若單幀 Disparity 突變量超過物理極限（馬達無法在一格內移動那麼快），判定為噪訊，予以剔除並沿用前一影格數據。

## 4. 動態對焦範圍限制演算法 (Dynamic Search Range Limiter)
*   **目的：** 反差對焦（Contrast AF）全行程掃描（0 至 1023 DAC）耗時太長。系統必須在掃描前收窄範圍。
*   **演算法機制：**
    *   當 ToF/Laser 輸出有效距離 $d$ 時，轉換為屈光度 $D = 1000/d$。
    *   利用 Diopter Table 查出對應馬達位置 $DAC_{estimate}$。
    *   **限幅計算：**
        $$DAC_{min} = DAC_{estimate} - \Delta DAC_{error}$$
        $$DAC_{max} = DAC_{estimate} + \Delta DAC_{error}$$
    *   反差對焦只在 $[DAC_{min}, DAC_{max}]$ 區間進行爬山掃描（通常寬度僅約 50-100 DAC），使反差對焦速度提升 10 倍。
